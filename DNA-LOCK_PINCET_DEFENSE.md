# DNA-Lock & Pincet Defense

실용 프롬프트 엔지니어링 기술 명세서 v1.0
독학 개발 · 모바일 환경 · 2026.04

## 1. 개요 (Overview)

본 문서는 추가 학습(Fine-tuning)이나 전용 하드웨어 없이, 프롬프트 구조만으로 AI 이미지 생성의 캐릭터 일관성을 높이고 LLM 보안을 강화하는 두 가지 실용 기법을 기술한다. 두 기법 모두 수만 회의 실전 반복을 통해 도출된 경험 기반 최적화 방법론이다.

## 2. DNA-Lock: 텍스트 기반 캐릭터 고정 기술

### 2.1 개발 배경

이미지 생성 AI(Nano Banana, Grok, Veo, Kling 등)는 여러 장의 레퍼런스 이미지를 입력받아야 캐릭터 일관성을 유지한다. 이 문제를 해결하기 위해 멀티앵글 통합 시트 1장 + 수치 앵커 텍스트의 조합을 실험적으로 개발했다.

### 2.2 워크플로우

- **Step 1** 원하는 캐릭터 이미지를 생성한다.
- **Step 2** 생성된 이미지를 AI에게 해부학적·수학적으로 분석시킨다 (골격 비율, 각도, 색상값 등).
- **Step 3** 분석 데이터 전체 중 '변하지 않는 핵심 수치'만 20~30%로 압축·요약한다.
- **Step 4** 압축된 DNA 텍스트를 프롬프트 최상단(Anchor)에 배치한다.
- **Step 5** 하단에 씬/의상/배경 등 가변 요소를 기술한다.

### 2.3 핵심 메커니즘: 3:7 Hybrid Structure

프롬프트 전체를 두 레이어로 구성한다.

- **Base Layer (70~80%)** 멀티앵글 통합 시트(정면/측면/후면 등 다각도를 1장에 압축)를 레퍼런스로 제공. 체감상 캐릭터 유지율 약 80% 달성.
- **DNA-Lock Layer (20~30%)** 분석된 해부학적 수치 중 핵심만 압축한 텍스트 앵커. 시트와 결합 시 유지율이 육안으로 구분 가능한 수준으로 향상됨. 전체 데이터의 30%를 초과하면 오히려 AI 연산에 충돌이 발생하므로 20~30%를 임계값으로 설정.

### 2.4 왜 작동하는가 (기술적 근거)

Diffusion 모델과 LLM은 공통적으로 프롬프트 앞부분에 더 높은 어텐션 가중치를 부여하는 경향이 있다. 또한 추상적 묘사보다 수치화된 구체적 파라미터가 모델의 확률 분포를 좁히는 효과가 있다. DNA-Lock은 이 두 특성을 동시에 활용한다.

### 2.5 DNA-Lock 예시 구조 (Scarlett)

실제 적용된 DNA-Lock 프롬프트의 구조를 간략히 나타내면, 상단 앵커에는 변하지 않는 핵심 수치만, 하단에는 씬별 가변 요소를 배치한다.

## 3. Pincet Defense: 가변형 프롬프트 보안 레이어

### 3.1 개발 배경 및 원리

DNA-Lock의 '핵심만 앵커로 고정한다'는 원리를 보안 영역에 응용한 기법이다. 기존의 일괄적 가드레일 방식이 AI 전체 성능을 저하시키는 문제를 해결하기 위해, 위험 신호가 감지될 때만 선택적으로 개입하는 구조를 설계했다.

### 3.2 3-Layer 구조

**Layer 1 — Identity Anchor (정체성 고정)**
시스템 프롬프트 최상단에 AI의 핵심 정체성(역할, 원칙)을 고정한다. 하위의 어떤 역할 부여나 가스라이팅 시도보다 우선순위(Priority 0)를 갖도록 명시한다.

**Layer 2 — Pincet Trigger (선택적 개입)**
평상시에는 AI의 추론 능력을 100% 개방한다. 특정 위험 키워드나 논리 패턴이 감지될 때만 보안 프로토콜이 활성화된다. 결과적으로 일반 대화의 품질 저하 없이 위험 상황에만 집중 대응한다.

**Layer 3 — Recursive Check (자가 검토 루프)**
답변 생성 직전, 해당 답변이 Layer 1의 정체성 원칙과 충돌하는지 스스로 검토하도록 유도한다. 가스라이팅으로 판단이 흐려지더라도 최종 출력 직전 앵커가 재활성화되는 구조다.

## 4. 확장 응용 가능성

DNA-Lock의 '핵심 앵커 + 가변 레이어' 구조는 이미지 생성과 보안 외에도 다양한 영역에 적용할 수 있다.

- **영상 제작**: 프레임 간 캐릭터 일관성 유지 (Grok, Kling, Veo 등)
- **IP 관리**: 캐릭터 IP를 수치화된 정체성으로 다양한 툴 간 이식 가능
- **페르소나**: LLM 페르소나 고정 — 챗봇의 역할·어조를 일관되게 유지
- **보안**: 커스텀 AI 에이전트 보안 레이어 설계

## 5. 결론

두 기법 모두 별도의 학습 데이터, GPU, 서버 비용 없이 프롬프트 구조 설계만으로 실용적인 효과를 달성한다. 독학, 모바일 환경, 수만 회의 반복 실험을 통해 경험적으로 도출된 방법론이며, 현재까지 동일한 구조를 기술한 공개 논문이나 문서는 확인되지 않았다.

기술의 가치는 '완벽함'이 아니라 '비용 대비 효과'에 있다. LoRA 없이 육안상 동일한 캐릭터를 유지하고, 추가 인프라 없이 일반적인 가스라이팅 공격의 대부분을 방어한다는 점에서 실용적 독창성을 갖는다.
